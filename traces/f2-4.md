# Trace: `f2(4)` — variation on Form A Q1B (2024)

```c
void f2(int x) {
    printf("%d\n", x);     // PRINT A (entry)
    if (x == 2) return;    // base case
    f2(x - 1);
    printf("%d\n", x);     // PRINT B (after 1st recursive call)
    f2(x - 1);
    printf("%d\n", x);     // PRINT C (after 2nd recursive call)
}
```

## Behavior summary

- `f2(2)` produces 1 line (just print A, then return).
- `f2(3)` produces 5 lines: A, [f2(2)=1], B, [f2(2)=1], C.
- `f2(4)` produces 13 lines: A, [f2(3)=5], B, [f2(3)=5], C.

## Key insight

Unlike `mystery`, non-base cases ALSO print here (A, B, C all execute on the way down/up).
So both internal nodes and leaves contribute to output.
